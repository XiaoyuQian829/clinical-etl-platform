## 已知问题 & 修复记录

| 问题 | 根因 | 修复 |
|---|---|---|
| Pydantic `model_config` import 失败 | pydantic v2 用 `ConfigDict` 不是 `model_config` | 三个 model 文件全部改为 `from pydantic import ConfigDict` |
| patients / admissions NaN 写入 PostgreSQL 报错 | pandas NaN/NaT 传入 pg 报 `DatatypeMismatch` | `etl/load.py` 加 `_clean_record()` 函数，NaN→None，NaT.isoformat()=="NaT"→None |
| passlib bcrypt 在 Python 3.14 报错 | bcrypt 库 API 不兼容 | 改用 `sha256_crypt` scheme |
| DBT schema 变成 `public_clean` 而非 `clean` | dbt 默认 prefix target.schema | 加 `dbt/macros/generate_schema_name.sql` macro |
| DBT `clean_diagnoses.sql` 引用不存在的列 | `raw.diagnoses` 没有 `icd_description` | 改为 `LEFT JOIN raw.icd_reference` 取 `long_title` |
| `research.cohort` 的 `primary_diagnosis_desc` 为空 | `clean_diagnoses` 之前写 `null::varchar` | 加载 `d_icd_diagnoses.csv` 到 `raw.icd_reference`，DBT JOIN 后 100% 覆盖 |
| `raw.diagnoses` 多次 pipeline 运行后行数翻倍 | serial PK 无 natural key 约束，ON CONFLICT 无效 | 加 `UNIQUE(subject_id, hadm_id, seq_num, icd_code, icd_version)` 约束 |
| audit log 写入失败（`:detail::jsonb` 语法错误） | SQLAlchemy 参数 `:x` 与 pg `::` 转型冲突 | 改为 `CAST(:detail AS jsonb)` |
| DBT research 模型创建了 `research.research_cohort` 而非 `research.cohort` | model 文件名带 `research_` 前缀 | 在 config 里加 `alias='cohort'` / `alias='outcomes'` |
